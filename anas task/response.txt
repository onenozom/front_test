## Architecture Overview

### `App.js`
- Manages the main state:
  - selected test type
  - live session timer
  - `isRunning`
  - active camera map
- Uses a `Map` for camera state so unchanged cameras preserve object references.
- Mocks a global WebSocket heartbeat and live rep updates.
- Starts the live session with a dynamic number of cameras (up to 10).
- Controls the 120-second countdown timer.
- Renders either the intro panel or the responsive `TraineeGrid`.

### `components/TraineeGrid.js`
- Receives `cameras` array.
- Chooses the Tailwind grid layout based on camera count:
  - `1 cam` → full width
  - `2-4 cams` → 2 columns
  - `5-9 cams` → responsive 3-column layout
  - `10 cams` → optimized `5` columns on large screens
- Keeps layout responsive with `grid` + `gap-4`.

### `components/TraineeCard.js`
- Handles each trainee tile individually.
- Uses `React.memo` with custom comparison to avoid re-rendering unless that trainee’s own props change.
- Uses `useRef` for rep count DOM updates so the numeric counter can update in-place.
- Uses `useMemo` for border/status style selection.
- Includes a styled video placeholder ready for integration with real stream packages.

## Performance Isolation

### How it is achieved
- `cameraMap` is stored as a `Map` so unchanged camera objects keep the same reference.
- `cameraList` is derived from the map, and only changed entries cause updated card props.
- `TraineeCard` is wrapped in `memo(...)` and only re-renders when:
  - `id`
  - `label`
  - `repCount`
  - `status`
  change.
- `useRef` is used inside card for rep counter DOM updates to reduce render pressure.
- Global timer and session state live in `App.js`, while per-card live updates stay isolated.

## Architecture Explanation بالعربية

### نظرة على المشروع
- `App.js` هو المشغل الرئيسي.
- `TraineeGrid.js` يبني شبكة متجاوبة حسب عدد الكاميرات.
- `TraineeCard.js` يعرض كل بطاقة متدرب بشكل مستقل.

### كيف تعمل الشبكة الديناميكية
- عندما يكون عدد الكاميرات 1، البطاقة تأخذ العرض الكامل.
- 2-4 كاميرات تصبح شبكة من عمودين.
- 5-9 كاميرات تتحول إلى شبكة ثلاثية الأعمدة على الشاشات الكبيرة.
- 10 كاميرات تستخدم تنسيقًا أكبر مع توزيع مناسب على الأعمدة.

### كيف يتم عزل الأداء
- يخزن `App.js` حالة الكاميرات في `Map` لكي لا يتغير مرجع الكائنات غير المعدلة.
- `TraineeCard` يستخدم `React.memo` حتى لا يعيد الرسم إلا إذا تغيرت بيانات تلك البطاقة فقط.
- يتم تحديث عدد التكرارات باستخدام `ref` في البطاقة، وهذا يقلل من الضغط الناتج عن التحديثات المتكررة.
- لذلك التحديث في كاميرا واحدة لا يعيد رسم الشبكة بأكملها.
